# Add Favicon Support

## Problem

The browser requests `/favicon.ico` automatically but returns 404 because:
1. No favicon file exists in the assets
2. No `<link rel="icon">` tag exists in the HTML templates

## Solution

Add favicon support by:
1. Creating a favicon.svg in example/assets/ (reuse the logo design)
2. Adding `<link rel="icon">` tag to all HTML templates
3. The existing asset copying mechanism will handle copying to public/

## Implementation Plan

### Task 1: Create favicon.svg
**Skill**: `go-writer`

Create `example/assets/favicon.svg` using the same design as `logo.svg` (blue circle with "QS").

**File**: `example/assets/favicon.svg`

### Task 2: Add favicon link to base.html template
**Skill**: `tdd-workflow`, `go-tester`, `go-writer`

1. Write test in `internal/renderer/renderer_test.go` verifying favicon link is rendered
2. Update `internal/renderer/templates/base.html` to add:
   ```html
   <link rel="icon" href="/favicon.svg" type="image/svg+xml">
   ```
   After the stylesheet link in `<head>`

### Task 3: Add favicon link to all other templates
**Skill**: `go-writer`

Update the following templates with the same favicon link:
- `internal/renderer/templates/home.html`
- `internal/renderer/templates/page.html`
- `internal/renderer/templates/blog_list.html`
- `internal/renderer/templates/blog_post.html`
- `internal/renderer/templates/404.html`

### Task 4: Verify with Playwright
**Skill**: `webapp-testing`

1. Build and serve the example site
2. Navigate to homepage
3. Verify no 404 errors for favicon
4. Take screenshot confirming favicon appears in browser tab

## Acceptance Criteria

1. `example/assets/favicon.svg` exists
2. All templates include `<link rel="icon" href="/favicon.svg" type="image/svg+xml">`
3. Favicon is copied to `public/` when building
4. No 404 error for favicon when browsing the site
5. Test covers favicon link rendering
